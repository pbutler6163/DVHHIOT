Apache Camel Example using Servlet REST & Web Container

There is a provider REST service that supports the following operations

  • GET /rest/provider/{user} - to view a user's score
  • GET /rest/provider/ - to view all users scores
  • PUT /rest/provider - to update a user's score
The view operations are HTTP GET, and update is using HTTP PUT. From a web browser you can access the first two services using the following links
  • rest/provider/{user} - to view the score of the user
  • rest/provider/ - to list all score
From the command shell you can use curl to access the service as shown below: 
    curl -X GET -H "Accept: application/json" http://localhost:8080/providerservice/rest/provider/{user}
    curl -X GET -H "Accept: application/json" http://localhost:8080/providerservice/rest/provider
    curl -X PUT -d "{ \"score\": 23 , \"userName\": \"Dr. Who\"}" -H "Accept: application/json" http://localhost:8080/rest/provider
Notice when trying to get provider with id 789 then there is a built-in delay of 0.5-2 seconds, to simulate some slowness in the service.

This example offers an API browser using Swagger which is accessible from the following link

  • api-docs - overview of the apis from the REST services
  • api-docs/provider - api of the provider REST service

To use the swagger ui, follow this link. Replace the URL value with this one to access the REST local resources (http://localhost:8080/api-docs

  • swagger - swagger ui page
